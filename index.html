WHY GITHUB FOR AI

AI時代にGitHubが要る理由

試行錯誤に強い

  • AI提案は一発で当たらないことが多い
  • 動いた版／壊れた版を履歴で比較できる
  • 原因追跡・ロールバックが容易

共有と再利用

  • レビュー・Issueで知見が貯まる
  • 後から来る自分・仲間が理解しやすい
  • 自動化（CI）にAIを組み込みやすい
本質4/17
DIFFERENCE

Git・GitHub・GitHub Copilotの違い

  • Git：ローカルで履歴管理・バージョン管理を行うツール
  • GitHub：Gitのリポジトリをクラウドで共有・コラボできるサービス
  • GitHub Copilot：AIによるコード補完・生成を支援する拡張機能

ACCOUNT

GitHubアカウント作成手順

  1. 公式サイト（https://github.com/）にアクセス
  2. 「Sign up」からメールアドレス・パスワードを登録

BASIC OPS

GUIでの基本操作
（GitHub Pull Requests拡張）

  1. サイドバーのGitHubアイコンをクリック
  2. 「リポジトリ」から対象リポジトリを開く
  3. 「変更」タブでファイルを編集し、ステージング・コミット
  4. 「プルリクエスト」タブで新規PRを作成・送信
  5. レビューやマージもVS Code上で完結
  • コマンド不要、直感的な操作が可能
  • コミットやPRの履歴もGUIで確認できる

AI INTEGRATIONS

AIとの連携（例）

AIエージェント連携

  • リポジトリを読み込み、要約・質問回答
  • Issueの起票・優先度付けの補助
  • CIで要約・レビュー・リリースノート自動化
  • 例：Claude Code（リポジトリ全体をAIで解析・要約・コード生成）

GitHub Copilot

  • エディタ内で補完・生成（雛形/テスト補助）
  • Copilot Chat / for PRs で説明文生成
  • まずは個人学習用リポジトリで試すのが安全
  • 例：コードレビューもCopilotにお願いできる

参考：Gitのインストール

「git: command not found」になってしまったあなたへ

Windowsの場合

  1. 公式サイトにアクセス
    https://git-scm.com/
  2. 「Download for Windows」をクリック
    自動でインストーラーがダウンロードされる
  3. インストーラーを実行
    ダウンロードされた Git-x.x.x-64.exe をダブルクリック
  4. セットアップ画面の指示に従う
    デフォルトでOK

Macの場合

  1. ターミナルに git --version を入力し、Enter
    初めての場合、「Command Line Developer Toolsが必要です。」と出る
  2. 「インストール」をクリック
    数分でインストール完了します。

対応完了後に git --version でバージョンが表示されるか確認

参考：GitHubアカウント作成の詳細手順

  1. 公式サイトにアクセス
    https://github.com/
  2. 右上の「Sign up」をクリック
  3. メールアドレスを入力して「Continue」
  4. パスワードを設定して「Continue」
  5. ユーザー名を決めて「Continue」
  6. メール通知の設定（y/n）を選択して「Continue」
  7. ロボットでないことを証明（パズル等）
  8. 「Create account」をクリック
  9. メールに届く認証コードを入力
参考14/17

参考：GitHub 2段階認証の設定（推奨）

  1. GitHubにログイン後、右上のプロフィール画像をクリック
  2. 「Settings」を選択
  3. 左サイドバーの「Password and authentication」をクリック
  4. 「Two-factor authentication」セクションの「Enable two-factor authentication」をクリック
  5. 認証アプリ（Google Authenticator、Authy等）またはSMS認証を選択
  6. QRコードをスキャンまたは電話番号を入力
  7. 生成される6桁のコードを入力して確認
  8. リカバリーコードを安全な場所に保存
参考15/17

参考：VS Code GitHub拡張機能の詳細導入手順

  1. VS Codeを起動
  2. 左サイドバーの拡張機能アイコン（四角形が4つ重なったアイコン）をクリック
  3. 検索ボックスに「GitHub Pull Requests and Issues」と入力
  4. 検索結果から「GitHub Pull Requests and Issues」（発行者：GitHub）を見つける
  5. 「Install」ボタンをクリック
  6. インストール完了後、左サイドバーにGitHubアイコンが表示される
  7. GitHubアイコンをクリックして「Sign in to GitHub」をクリック
  8. ブラウザでGitHubログインを完了し、VS Codeへの認証を許可

参考：GitHub拡張機能の基本的な使い方

リポジトリのクローン

  1. VS CodeでCtrl+Shift+P（macOS: Cmd+Shift+P）
  2. 「Git: Clone」と入力して選択
  3. GitHubのリポジトリURLを入力
  4. 保存先フォルダを選択

変更のコミット・プッシュ

  1. ファイルを編集・保存
  2. 左サイドバーのGitアイコンをクリック
  3. 「Changes」で変更ファイルを確認
  4. 「+」でステージング、コミットメッセージを入力
  5. 「Commit」→「Sync Changes」でプッシュ
